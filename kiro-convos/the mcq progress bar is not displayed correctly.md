Here's a comprehensive breakdown of how the MCQ progress bar, attempts tracking, and lesson completion summary work:

## 1. MCQ Progress Bar (`MCQCardWidget.tsx`)

The progress bar shows **lesson-level** progress, not just objective-level:

- **Calculation**: It counts `totalCorrectSoFar` by filtering `Object.values(userAttempts)` for entries where `correct === true`.
- **Estimated total**: `totalObjectives * totalQuestions` (questions in the current batch) — or falls back to `totalObjectives * 2` if the product is 0.
- **Display**: A `<Progress>` component showing `{totalCorrectSoFar}/{estimatedTotal} questions` with a visual bar.

Additionally, there are **Badge indicators** for:
- Objective position: `Objective X/Y`
- Question position within objective: `Question X/Y`
- Attempt status: Shows "Solved in N attempt(s)" or "Attempt N" depending on correct/incorrect state.

The progress bar is an *estimate* because questions are generated per-objective (2 per objective via `QUESTIONS_PER_OBJECTIVE = 2` in nodes.ts), and future batches haven't been generated yet.

## 2. Attempts Tracking

Attempts are tracked **per-question**, not per-session:

- **State shape** (`UserAttempts`): `Record<string, { attempts: number; correct: boolean }>` keyed by question ID.
- **State reducer** in `state.ts`: Uses `(prev, next) => ({ ...prev, ...next })` — a merge reducer that accumulates across questions.
- **Evaluator node** (`evaluatorNode` in nodes.ts): On each submission, it increments the attempt count for that specific question: `prev.attempts + 1`. The `correct` field is set to `prev.correct || isCorrect` (sticky — once correct, stays correct).
- **Retry flow**: On incorrect answer, routing loops back to the evaluator (same question, same interrupt). The hint is displayed, user picks a new answer, and attempts increment again. There's no penalty — all attempts are tracked but the question is marked correct as soon as they get it right.
- **Client persistence**: The `useLearningAgent` hook also fires `recordAttempt()` to the `/api/sessions` endpoint to persist each attempt in the database.

## 3. Lesson Completion Summary (`summaryNode` + `LessonSummary.tsx`)

**Data computation** (in `summaryNode` in nodes.ts):
- `totalQuestions`: Count of all question IDs in `userAttempts`
- `totalCorrect`: Count of questions where `attempts[id].correct === true`
- `totalAttempts`: Sum of all `attempts[id].attempts` across all questions (includes retries)
- `perObjective`: Groups questions by their `obj{index}-` prefix in the question ID, counts correct/total per objective
- `studyTips`: Generated by the LLM (Gemini/Groq) using the `StudyTipsSchema` structured output, personalized to the learner's performance profile. Falls back to generic tips if the LLM call fails.

**Display** (`LessonSummary.tsx`):
- Overall accuracy percentage: `Math.round((totalCorrect / totalQuestions) * 100)`
- Progress bar showing correct/total
- Per-objective breakdown list showing `{correct}/{total}` for each objective
- Study tips as a bulleted list
- "Start a new lesson" button that calls `reset()`

**Session persistence**: When the summary arrives, `useLearningAgent` automatically calls `completeSession()` to save the final stats (including computed accuracy) to the database and refreshes the session history list.
